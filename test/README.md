这是一个测试用例的文件夹，包含一些模拟的对话的尝试来测试LLM和Agent的Bug。

主要测试方式包括验证功能和客户的需求。
1. 初始化项目：用户开启对话来新建项目，描述需求，CA反复确认需求，并细化（如果人才库中没人，或者新建了这个项目，是不需要完全进行人才匹配的，新建好了项目后，请说：好的，我已经成功创建了项目，已经保存好。）
2. 修改项目：比如修改项目名称、position、（这些其实可以有两种方式：用户直接在项目管理中修改+跟CA说自己要如何去改）
3. 触发行业研究：在新建项目的时候，这个按钮是灰色不可以点击的。当项目创建完成后，可以进行触发行业研究，触发RA进行工作，并返回报告。
4. 触发人才评估：当对CA说，帮我从人才库中找到一些匹配的人，然后CA会直接从人才库中获取到相对应的一些人，然后触发EA进行评估，拿到结果并返回。
